## Typography

Unchanged from Phase 1 (source: 01-UI-SPEC.md), with Phase 4 weight consolidation applied:

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display heading | 28px | 600 | 1.2 | Playfair Display |
| Section heading | 20px | 600 | 1.3 | DM Sans |
| Body | 16px | 400 | 1.5 | DM Sans |
| Label / UI text + Mono / data | 14px | 400 | 1.4 | DM Sans / DM Mono |

**Weight consolidation (Phase 4 fix):** Exactly 2 weights in use — 400 (regular) and 600 (semibold). Weight 500 is removed; all elements previously at 500 use 400. Weight 700 is removed; Playfair Display display headings use 600.

**Size consolidation (Phase 4 fix):** Exactly 4 sizes — 28px, 20px, 16px, 14px. Size 13px is removed; all elements previously at 13px (DM Mono data, file sizes, timestamps, framework badge labels) use 14px. Size 12px is removed; SHA-256 hash display uses 14px DM Mono.

Phase 4 additions:
- Compliance posture percentage (% met): DM Mono 28px/600, color matched to semantic status (ok/warn/err) — large, prominent stat display
- SHA-256 hash value: DM Mono 14px/400, color navy-mid (#3A5270), wraps within a `<code>` element — full 64 chars must be visible
- File size display: DM Mono 14px/400, color navy-mid — displayed alongside original filename in evidence list
- Attestation timestamp: DM Mono 14px/400, color navy-mid — "Attested [date] by [name]" pattern
- Framework badge label: DM Sans 14px/400 — compact badge text for regulatory framework chips

## Interaction Contracts

### Form Validation Timing

Inherits Phase 1 rules:
- Required field validation: on blur
- Format validation: on blur
- File type and size validation: immediately on file selection (before SHA-256 compute)
- SHA-256 computation: triggered immediately after file selection; submit button disabled until complete
- All fields re-validated: on submit before API call
- No live validation on first interaction

### Loading States

| Action | Loading behavior |
|--------|-----------------|
| Save Obligation | "Save Obligation" button shows spinner, text stays, button disabled, no page shift |
| Update Obligation | "Update Obligation" button shows spinner, disabled |
| Submit Attestation | "Submit Attestation" button shows spinner, disabled; all form inputs disabled |
| Upload Evidence | "Upload Evidence" button shows spinner + "Uploading..." text; progress state visible in drop zone |
| SHA-256 computation | Drop zone shows "Computing checksum..." with inline spinner; submit button remains disabled |
| Obligations list load | Table shows 5 skeleton rows (48px each, gray shimmer) |
| Compliance dashboard load | 3 stat cards show gray shimmer placeholder rectangles |
| Evidence download | Download button shows spinner + "Verifying..." until integrity check completes |

### Success Transitions

| Action | Behavior |
|--------|---------|
| Obligation saved (new) | Redirect to `/compliance/obligations/[id]` (detail page), success toast |
| Obligation updated | Redirect to `/compliance/obligations/[id]`, success toast |
| Attestation submitted | Redirect to `/compliance/obligations/[id]` (attestation history section auto-scrolled), success toast |
| Evidence uploaded | Redirect to `/compliance/obligations/[id]` (evidence section), success toast |
| Obligation status changed (inline select) | Status badge updates in place, no page reload; if requires dialog → confirmation dialog then update, success toast |
| Evidence download (integrity OK) | File download initiated, success toast |
| Evidence download (integrity fail) | Download blocked, destructive persistent toast |

### Filter Interaction

- Filters are client-side (TanStack Table `setColumnFilters`) — no network round-trip
- Instant response on selection change
- Active filter count displayed in subtitle: "Filtered: [N]" updates reactively
- Clearing filters: individual clear per Select, or "Clear filters" button resets all at once

### Keyboard Navigation

Inherits Phase 1 and Phase 3 rules. Phase 4 additions:

- Obligations table: Tab to filter bar, then Tab to table. Arrow keys navigate rows. Enter navigates to obligation detail.
- Evidence drop zone: Tab reaches drop zone. Enter/Space opens file browser. Keyboard equivalent to drag-and-drop.
- Attestation radio cards: Tab between radio cards. Arrow keys (Up/Down) navigate options. Space selects focused card.
- Attestation form: Tab order — status radio group → notes textarea → cancel → submit.
- Obligation status select (inline): Tab reaches select trigger. Enter/Space opens options. Arrow keys navigate. Enter selects. Escape closes without change.

## Accessibility

Inherits all Phase 1 and Phase 3 accessibility rules. Phase 4 additions:

### ARIA Roles and Attributes

| Element | ARIA attributes |
|---------|----------------|
| Evidence drop zone | `role="button"`, `aria-label="Upload evidence file. Drop a file here or press Enter to browse."`, `aria-live="polite"` for state changes |
| SHA-256 computation status | `aria-live="polite"` — announces "Computing checksum..." and "Checksum complete" |
| Attestation radio group | `role="radiogroup"`, `aria-label="Select compliance status"` |
| Attestation radio card | `role="radio"`, `aria-checked="true/false"`, `aria-label="[status label]: [description]"` |
| Obligation status badge | `aria-label="Compliance status: [status display text]"` |
| Framework badge | `aria-label="Regulatory framework: [framework display name]"` |
| Evidence integrity icon | `aria-label="Integrity status: [Verified / Mismatch / Not verified]"` |
| Obligations table | `role="grid"`, `aria-label="Compliance obligations"` |
| Overdue row | `aria-label="[Row content] — Overdue"` — row description includes overdue state |
| Attestation history list | `aria-label="Attestation history, [N] records"`, `aria-live="polite"` (new attestations announced) |
| SHA-256 hash display | `aria-label="SHA-256 checksum: [full hash value]"` — Tooltip provides full hash; truncated display is decorative |
| Digital signature notice | `role="note"`, `aria-label="Attestation will be recorded in the immutable audit trail"` |
| View obligation button (icon-only) | `aria-label="View obligation: [obligation title]"` + Tooltip: "View obligation" |
| Edit obligation button (icon-only) | `aria-label="Edit obligation: [obligation title]"` + Tooltip: "Edit obligation" |

### Color Independence

Compliance status must never be communicated by color alone. Always pair color with:
- Status badge text label (Compliant / Partially Compliant / Non-Compliant / Overdue / Pending / Waived)
- Table row left-border accent (overdue = err, expiring = warn) is supplementary only — not sole indicator
- Stat card values include descriptive text beneath the number

Evidence integrity must pair the color-coded icon with text ("Verified" / "Mismatch" / "Not verified").

### Focus Management

- On obligation save (redirect): focus moves to `<h1>` of detail page
- On attestation save (redirect + scroll): focus moves to attestation history section heading
- On evidence upload (redirect): focus moves to evidence section heading
- On dialog open (destructive status change): focus trapped in dialog
- On dialog cancel/confirm: focus returns to trigger element (status select)
- On evidence drop zone file selection: focus remains on drop zone; SHA-256 progress announced via `aria-live`
