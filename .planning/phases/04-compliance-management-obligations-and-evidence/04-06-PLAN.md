  <how-to-verify>
    Start the development server: `npm run dev`

    **Test 1: Navigation**
    - Log in as a compliance-officer user (or admin)
    - Verify "Compliance" appears in the sidebar navigation
    - Log in as a dept-head user
    - Verify "Compliance" does NOT appear in the sidebar (or that /compliance redirects to /dashboard)

    **Test 2: Compliance Posture Dashboard**
    - Navigate to http://localhost:3000/compliance
    - Page should load without errors (no 500, no "force-dynamic" warnings)
    - 3 stat cards visible: "COMPLIANCE RATE", "OVERDUE", "EXPIRING SOON"
    - If no obligations exist: empty state shows "No obligations recorded" with "Add Obligation" button
    - If obligations exist: stats reflect correct counts

    **Test 3: Create Obligation**
    - Click "Add Obligation"
    - Form at /compliance/obligations/new should load
    - Fill in: Title, select a Framework (e.g., PECOGA), select an Owner, set Due Date
    - Change framework to "Other" — verify "Regulation reference" field becomes required (try submitting without it)
    - Submit the form — obligation should be created and redirect to detail page
    - Verify obligation appears in the obligations list at /compliance/obligations

    **Test 4: Evidence Upload**
    - On the obligation detail page, click "Upload Evidence"
    - Drop zone should show at /compliance/obligations/[id]/evidence/upload
    - Select a PDF or DOCX file under 25 MB
    - Drop zone should show: file name, file size, "Computing checksum..." then the full 64-char SHA-256 hash
    - Submit button should be disabled until hash computation completes
    - Upload the file — should redirect back to detail page
    - Evidence file appears in the Evidence Files section with filename, size, upload date

    **Test 5: Evidence Download with Integrity Check**
    - On the obligation detail page, click "Download File" next to the uploaded evidence
    - File should download (browser download dialog appears) — NOT a 409 error
    - The download URL should be /api/compliance/evidence/[evidence-id] (check browser network tab)
    - Verify file content is intact after download

    **Test 6: Attestation**
    - On the obligation detail page, click "Attest"
    - Attestation form at /compliance/obligations/[id]/attest should load
    - Three radio cards visible: Compliant, Partially Compliant, Non-Compliant
    - Digital signature notice Alert visible ("This attestation will be signed")
    - Select "Compliant", optionally add notes, submit
    - Redirect to detail page — attestation appears in Attestation History section
    - Obligation status badge updates to "Compliant"
    - Attest again with "Partially Compliant" — both attestations visible in history (newest first)

    **Test 7: Obligations List Filtering**
    - Navigate to /compliance/obligations
    - Obligations table visible with TanStack Table
    - Framework filter: select "PECOGA" — table updates instantly without page reload
    - Status filter: select "Pending" — table filters to pending obligations
    - "Clear filters" button appears when filters active; clicking resets to show all
    - Overdue rows (past due_date, not compliant) should show left red border

    **Test 8: Escalation Route (Admin only)**
    - Open a terminal and run:
      ```bash
      curl -X POST http://localhost:3000/api/compliance/escalate \
        -H "x-cron-secret: [your CRON_SECRET from .env.local]"
      ```
    - Should return JSON: `{"success":true,"obligationsFound":N,"emailsSent":N,"emailsSkipped":N}`
    - Run without the CRON_SECRET header — should return 401 Unauthorized

    **Test 9: Role-Based Access**
    - Log in as a risk-officer
    - Verify /compliance dashboard is accessible (view access per D-32)
    - Verify /compliance/obligations/new redirects to /compliance/obligations (no write access)
    - Log in as board-member
    - Verify /compliance/obligations/[id]/attest redirects (no attest access)
