## Typography

Unchanged from Phase 4 (source: 04-UI-SPEC.md). Exactly 4 sizes, exactly 2 weights:

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display heading | 28px | 600 | 1.2 | Playfair Display |
| Section heading | 20px | 600 | 1.3 | DM Sans |
| Body | 16px | 400 | 1.5 | DM Sans |
| Label / UI text + Mono / data | 14px | 400 | 1.4 | DM Sans / DM Mono |

**Exactly 2 weights only:** 400 (regular) and 600 (semibold). No 500, no 700.
**Exactly 4 sizes only:** 28px, 20px, 16px, 14px. No 13px, no 12px.

Phase 5 additions to typography usage:
- Resolution number display ("RES-001"): DM Mono 14px/600, color navy-900 — semibold to signal a formal record identifier
- Meeting date/time display: DM Mono 14px/400, color navy-mid — "DD MMM YYYY at HH:MM" format
- Vote tally text (e.g. "For: 8 / Against: 2 / Abstain: 1"): DM Mono 14px/400, color navy-mid
- "CLOSED" immutability banner text: DM Sans 14px/600, color err — uppercase, semibold to signal locked state
- Board dashboard stat values: DM Mono 28px/600, color matched to semantic status (ok/warn/err) — identical to ComplianceStatCard pattern

| --c-gold | #C8A44A | `gold` | Accent (10%) |
| --c-gold-pale | #F3E2BC | `gold-pale` | Accent tint |
| --c-ok | #27AE60 | `ok` | Passed / success |
| --c-warn | #E67E22 | `warn` | In Progress / tabled / warning |
| --c-err | #E74C3C | `err` | Rejected / error / overdue |

**60/30/10 split:**
- 60% dominant: `paper` (#F3F7FD) — page background
- 30% secondary: white (#FFFFFF) — cards, table surface, filter bar card, dialog backgrounds
- 10% accent: `gold` (#C8A44A) — reserved elements listed below

**Accent (#C8A44A) is reserved exclusively for:**
- "Save Meeting" / "Record Resolution" / "Save Action Item" / "Upload Document" primary CTA buttons
- "Close Meeting" destructive CTA button uses `err` (#E74C3C), NOT gold — closure is destructive, not primary creation
- Active navigation item indicator (inherited from Phase 1)
- Focus ring on all interactive elements (2px solid gold, 2px offset — inherited)
- Active filter `Select` ring (`ring-2 ring-gold/50` — inherited from Phase 3 filter bar pattern)
- Board pack upload drop zone drag-over active border (`border-gold bg-gold-pale/20` — mirrors Phase 4 pattern)

### Color Independence

Meeting status, resolution outcome, and action item status must never be communicated by color alone. Always pair color with:
- Badge text label (Scheduled / In Progress / Closed / Passed / Rejected / Tabled / Open / etc.)
- Table row left-border accent (overdue rows = err) is supplementary only — not sole indicator
- Stat card values include descriptive text beneath the number

Closed meeting immutability banner uses `Lock` icon + text in addition to err color.

### Focus Management

- On meeting save (redirect): focus moves to `<h1>` of meeting detail page
- On document upload (redirect): focus moves to Documents tab trigger (tab auto-activated)
- On resolution recorded (redirect): focus moves to Resolutions tab trigger (tab auto-activated)
- On action item saved (redirect): focus moves to Actions tab trigger (tab auto-activated)
- On meeting closed (redirect): focus moves to `<h1>`, closed banner announced via `aria-live="assertive"`
- On close meeting dialog open: focus trapped in dialog (`role="alertdialog"`)
- On dialog cancel: focus returns to "Close Meeting" trigger button
- On dialog confirm: focus moves to `<h1>` after redirect
